# AddFuck 虚拟机

AddFuck 是一个简单的基于内存操作的虚拟机，支持基本的加法运算、条件加法、间接加法以及输入输出操作。

## 功能特性

- **基本加法运算**：支持立即数加法
- **条件加法**：当进位标志为1时执行加法操作
- **间接加法**：通过内存地址间接获取操作数
- **输入操作**：从用户输入读取数据到指定内存缓冲区
- **输出操作**：从指定内存缓冲区输出数据
- **UTF-8 编码支持**：正确处理 UTF-8 字符

## 操作码说明

| 操作码 | 功能描述 | 参数1 | 参数2 |
|-------|---------|------|------|
| A     | 立即数加法 | 目标内存地址 | 立即数值 |
| CA    | 条件加法 | 目标内存地址 | 源内存地址 |
| IA    | 间接加法 | 目标内存地址 | 源内存地址（存储操作数地址） |
| I     | 输入操作 | 缓冲区起始地址 | 缓冲区长度 |
| O     | 输出操作 | 缓冲区起始地址 | 缓冲区长度 |

## 内存布局

- `mem[0]`：程序计数器（PC）
- `mem[1]`：进位标志（CF）
- `mem[2...]`：用户可用内存

## 使用方法

### 构建项目

```bash
mkdir -p build && cd build
cmake ..
make
```

### 运行程序

```bash
./addfuck <程序文件路径>
```

### 示例程序

#### 计算两数之和并输出

```
; 读取两个输入值
I 10 10
I 20 10

; 计算和
A 30 0
CA 30 10
CA 30 20

; 输出结果
O 30 10
```

## 指令格式

每行一条指令，格式为：

```
<操作码> <参数1> <参数2>
```

- 以 `;` 开头的行被视为注释
- 空行将被忽略

## 性能优化

项目已启用以下优化选项以减小可执行文件大小：

- `-O2`：优化级别 2
- `-s`：移除符号表和调试信息
- `-ffunction-sections -fdata-sections`：将每个函数和数据放在单独的节中
- `-Wl,--gc-sections`：移除未使用的节
